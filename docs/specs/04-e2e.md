# 04 — End-to-end generate-and-run

## Goal

Prove that `feathers new` produces a service that actually boots and responds on
`/health`. This is the acceptance-gate test for Phase 1.

## Inputs / Outputs

- Runs the full pipeline: YAML → generated service → `uv sync` → `uvicorn` → HTTP GET.
- Marked `@pytest.mark.slow`; opt-in via `pytest -m slow`.

## Test cases

- `test_generate_users_service_boots_and_healthchecks`:
  1. Render `demos/users.yaml` to a tmp dir.
  2. Run `uv sync` inside the generated dir. Must exit 0.
  3. Launch `uv run uvicorn <pkg>.main:app --port <free>` as a subprocess.
  4. Poll `GET http://localhost:<port>/health` up to 10s. Must return 200
     with JSON `{"status":"ok","service":"hello_users",...}`.
  5. Kill subprocess cleanly.

- `test_generated_service_has_platform_middleware`:
  - After render, the generated service's `src/<pkg>/core/platform.py` exists and
    defines `install_platform_middleware`.

- `test_generated_ci_workflow_valid_yaml`:
  - After render, `.github/workflows/ci.yml` exists and parses as valid YAML with
    `jobs.lint`, `jobs.test`, `jobs.build`.

## Acceptance

- Full pipeline completes in <60 seconds on a cold run (uv sync dominates).
- No network access required except what uv already does for dep resolution.
- Test runs in CI on every push (slow marker opt-in via the ci workflow).
